# Skill: generate-core-feature

## Responsibility
Generate the specification for the single most important feature of a SaaS product.

## Input
- `product_name`: Name of the SaaS product
- `value_proposition`: One-sentence value proposition
- `target_user`: Primary user persona

## Steps
1. Identify the core feature that directly delivers the value proposition.
2. Write a feature specification including: goal, user story, acceptance criteria, and API contract outline.
3. List data entities required.
4. Identify dependencies or prerequisites.

## Output
- Feature name
- User story
- Acceptance criteria (checklist)
- API contract outline
- Required data entities

## Notes
- The core feature must be the smallest unit that delivers the value proposition.
- Do not scope creep into secondary features.
